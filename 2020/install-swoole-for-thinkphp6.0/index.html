服务器上安装 TP6.0 和 Swoole 扩展

入门 2020-03-13 1371 字 3 分钟

1 linux源码编译安装php

wget下载php源码：

wget https://www.php.net/distributions/php-7.2.27.tar.gz

linux安装php依赖：

sudo yum install -y gcc gcc-c++  make zlib zlib-devel pcre pcre-devel  libjpeg libjpeg-devel libpng libpng-devel freetype freetype-devel libxml2 libxml2-devel glibc glibc-devel glib2 glib2-devel bzip2 bzip2-devel ncurses ncurses-devel curl curl-devel e2fsprogs e2fsprogs-devel krb5 krb5-devel openssl openssl-devel openldap openldap-devel nss_ldap openldap-clients openldap-servers

解压php源码：

tar -zxvf php-7.2.27.tar.gz

进入文件夹：

cd php-7.2.27

新建php安装目录：

sudo mkdir /usr/local/php

php源码文件夹中有一个可执行文件config，使用这个文件进行环境检测，生成 Makefile，为编译做准备：

./configure
--prefix=/usr/local/php
--with-config-file-path=/usr/local/php/etc
--enable-inline-optimization
--disable-debug
--disable-rpath
--enable-shared
--enable-soap
--with-xmlrpc
--with-openssl
--with-pcre-regex
--with-sqlite3
--with-zlib
--enable-bcmath
--with-iconv
--with-bz2
--enable-calendar
--with-curl
--with-cdb
--enable-dom
--enable-exif
--enable-fileinfo
--enable-filter
--with-pcre-dir
--enable-ftp
--with-gd
--with-openssl-dir
--with-jpeg-dir
--with-png-dir
--with-freetype-dir
--enable-gd-jis-conv
--with-gettext
--with-mhash
--enable-json
--enable-mbstring
--enable-mbregex
--enable-mbregex-backtrack
--with-libmbfl--with-onig
--with-mysqli=shared,mysqlnd
--with-pdo-mysql=shared,mysqlnd
--with-zlib-dir
--with-pdo-sqlite--enable-session
--enable-shmop--enable-simplexml
--enable-pcntl
--enable-sockets
--enable-sysvmsg
--enable-sysvsem
--enable-sysvshm
--enable-wddx
--with-libxml-dir
--enable-zip
--enable-mysqlnd-compression-support
--with-pear
--enable-opcache

WARNING：

1 先切到root用户，否则编译安装后无法创建相关文件夹

2 最好先进行环境清理 make clean

make是用来编译的，它从Makefile中读取指令，然后编译：

make

make install是用来安装的，它也从Makefile中读取指令，安装到指定的位置：

make install

报错：virtual memory exhausted: 无法分配内存 make: *** [ext/fileinfo/libmagic/apprentice.lo] 错误 1

原因：内存不足，无法完成编译（vps买不起贵的）

解决：设置虚拟内存

编译安装PHP组件出现错误提示内存不足：virtual memory exhausted Cannot allocate memory

查看内存使用情况：

free -m

创建目录：

sudo mkdir /opt/images/

创建一个2GB大小的文件：

sudo dd if=/dev/zero of=/opt/images/swap bs=1024 count=2048000

把创建的文件变成SWAP分区：

sudo mkswap /opt/images/swap

启用这个SWAP文件：

sudo swapon /opt/images/swap

查看SWAP是否生效：

free -m

已经生效，继续编译安装：

make && make install

完成后释放虚拟内存：

sudo swapoff swap

或者：

sudo swapoff -a

删除：

rm -f /opt/images/swap

将php添加到环境变量中方便使用：

vim /etc/profile.d/php.sh
export PATH=$PATH:/usr/local/php/bin

重新载入以生效：

source /etc/profile.d/php.sh

查看php配置文件应该放在哪个目录：

php --ini

把配置文件复制到该目录中：

cp php.ini-production /etc/php.ini

2 安装swoole模块

下载并解压：

#下载
wget http://pecl.php.net/get/swoole-4.4.15.tgz
#解压
tar -zxvf swoole-4.4.15.tgz 
#进目录
cd swoole-4.4.15/

发现没有config文件：

phpize

生成了config文件，继续构建编译安装：

#构建
./configure --enable-openssl  --enable-http2
#如果有多个PHP版本，需要指定php-config
./configure  --with-php-config=/usr/local/php/bin/php-config --enable-openssl  --enable-http2
#编译
make
#安装
make install

安装完成后显示安装地址：

Installing shared extensions:     /usr/local/lib/php/extensions/no-debug-non-zts-20170718/
Installing header files:          /usr/local/include/php/

将swoole添加到php中：

#打开php.ini文件
vi /usr/local/lib/php.ini
#写入swoole
extension=swoole.so

查看swoole模块是否已经添加：

php -m

测试：

php /swoole-4.4.15/examples/server/echo.php

没有报错，说明已经可以使用

3 php动态添加扩展

进入到扩展目录：

cd /php-7.2.27/ext

里面都是php扩展，要安装哪个扩展就进入该扩展的文件夹，例如curl：

cd curl/

这里是curl的源码，执行phpize（php添加扩展的外挂程序）：

phpize

在当前文件夹中生成了config文件，执行：

./configure
#切记一切make之前都要先进行make clean
make clean
make
make install

显示扩展文件安装路径：

Installing shared extensions:     /usr/local/lib/php/extensions/no-debug-non-zts-20170718/

最后在php.ini中添加扩展：

vim /usr/local/lib/php.ini
extension=curl.so

查看扩展相关信息：

php --ri curl

4 安装Composer

curl -sS https://getcomposer.org/installer | php
mv composer.phar /usr/local/bin/composer

由于众所周知的原因，国内使用composer速度感人，使用命令：

composer config -g repo.packagist composer https://packagist.phpcomposer.com

切换为国内镜像，当然速度比原先快一些但是速度依然感人。如果要恢复官方源，使用命令：

composer config -g --unset repos.packagist

5 使用Composer安装ThinkPHP6.0

创建项目目录：

mkdir swoole

设置权限777：

chmod -R 777 swoole

进入目录并安装开发版：

cd swoole
composer create-project topthink/think=6.0.x-dev tp

复制.env文件：

cp .example.env .env

测试运行：

php think run

浏览器访问 http://localhost:8000/

TP6安装完成！

6 安装TP6的swoole扩展

到项目根目录：

composer require topthink/think-swoole

安装好后在根目录使用命令：

php think swoole

如果显示启动了swoole服务就说明安装成功了 ！

　　以上就是使用源码安装php、swoole扩展、thinkPHP6.0和think-swoole插件的全过程，强烈推荐小白自己过一遍，对Linux操作能有一个大致的理解。
